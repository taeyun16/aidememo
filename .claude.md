# CLAUDE.md — Claude Code용 Wiki-Graph 프로젝트 가이드

## 프로젝트 개요

wg(Wiki-Graph)는 마크다운 기반 위키를 구조화된 knowledge graph로 변환하는 Rust CLI 도구입니다. LLM 워크플로우에 최적화되어 있습니다.

**작업 디렉터리**: `/Users/mixlink/dev/wg`

## 핵심 명령

```bash
# 빌드 (전체 또는 개별)
cargo build -p wg-core -p wg-cli
cargo check -p wg-cli  # 빠른 검증

# 빌드 에러 확인
cargo build 2>&1 | grep "^error"

# CLI 실행
./target/debug/wg --help
./target/debug/wg init ./my-wiki
./target/debug/wg entity list
```


## bpaf CLI 규칙 (매우 중요)

bpaf 0.9는 positional/command item이 **struct/tuple의 오른쪽**에 위치해야 합니다. 이 규칙을 위반하면 런타임에 패닉합니다.

**올바른 예**:
```rust
pub struct InitSub {
    pub no_ingest: bool,     // flag → 왼쪽
    pub wiki_root: PathBuf,  // positional → 오른쪽 (가장 오른쪽)
}

construct!(InitSub { no_ingest, wiki_root })  // struct 순서와 일치
```

**잘못된 예** (패닉 유발):
```rust
pub struct InitSub {
    pub wiki_root: PathBuf,   // positional인데 왼쪽에 있으면 패닉!
    pub no_ingest: bool,
}
```

`construct!` 배열 안에 `init::init_command()` 같은 `::` 경로 함수 호출도 불가합니다. 반드시 지역 변수를 사용하세요:
```rust
let init_cmd = init::init_command();
construct!([..., init_cmd, ...])
```

## 자주 보는 빌드 에러 처리

### 1. bpaf 패닉 (런타임)
```
bpaf usage BUG: all positional and command items must be placed in the right most position
```
→ struct 필드 순서 확인: positional은 오른쪽, flags는 왼쪽
→ `construct!` 인자 순서도 struct 순서와 일치하는지 확인

### 2. E0425: cannot find type
`WgError`, `Config`, `WikiGraph` 등이找不到 → `wg_core::` prefix로 import 확인
`wg_core` crate는 `crates/wg-core/`에 있으며 `lib.rs`에서 public re-export됨

### 3. redb 트랜잭션 에러
`Cannot start a read transaction inside a read transaction` → txn 중첩 불가, txn을 drop 후 재시작

### 4. strsim normalized_similarity 없음
`trigram::normalized_similarity`은 존재하지 않음 → `strsim::jaro_winkler(a, b)` 사용

## Ingest 파이프라인

`ingest_wiki(wiki_root, store, incremental)`:
1. `WalkDir`로 `.md` 파일 스캔
2. frontmatter 파싱 (entity type, tags, aliases)
3. `[[wikilink]]` 추출 → relation 후보
4. heading-anchored section 파싱 → fact 후보
5. `store.add_entity()`, `store.add_fact()`, `store.add_relation()` 호출
6. `stats.entities_added`, `relations_added`, `facts_added` 수집

## Config

`~/.wg/config.toml`에서 로드. 없으면 기본값 사용.
`Config.store.path`는 `String`이므로 `PathBuf::from()` 필요.
