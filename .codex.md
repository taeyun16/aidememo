# CODEX.md — OpenAI Codex용 Wiki-Graph 프로젝트 가이드

## 프로젝트 개요

wg(Wiki-Graph)는 마크다운 기반 위키를 구조화된 knowledge graph로 변환하는 Rust CLI 도구입니다.
**작업 디렉터리**: `/Users/mixlink/dev/wg`

## 빌드 명령

```bash
# 전체 빌드 확인
cargo check -p wg-core -p wg-cli

# 전체 빌드
cargo build -p wg-core -p wg-cli

# CLI 실행
./target/debug/wg --help
```

## 주요 코드 위치

| 목적 | 파일 |
|------|------|
| WikiGraph API | `crates/wg-core/src/lib.rs` |
| redb 저장소 | `crates/wg-core/src/store.rs` |
| Entity/Fact 타입 | `crates/wg-core/src/types.rs` |

| bpaf 명령 등록 | `crates/wg-cli/src/cmd/init.rs`, `watch.rs` |

## bpaf CLI 규칙 (중요)

`construct!` 배열 안에 `init::init_command()` 같은 `::` 경로 함수 호출 금지. 반드시:
```rust
let init_cmd = init::init_command();
construct!([..., init_cmd, ...])
```

positional 필드는 struct 오른쪽에 위치. `construct!` 인자 순서는 struct 필드 순서와 일치.

## 자주 보는 에러 Quick Fix

| 에러 | 해결 |
|------|------|
| `cannot find type WgError` | `wg_core::WgError`로 import 확인 |
| `no method parent on String` | `config.store.path`는 `String` → `PathBuf::from()` 변환 |
| `bpaf usage BUG` | struct 필드 순서에서 positional이 오른쪽인지 확인 |
| `E0505: borrow of moved value` | `config.search.default_limit` 등 move할 값을 클로저 전에 로컬 변수로 추출 |

## 프로젝트 규칙

- redb 트랜잭션: read txn과 write txn 중첩 불가
- `Store`는 `Arc<RwLock<Store>>`로 공유 (mutable borrow 문제 회피)
- `Config.store.path`는 `String` 타입
- `WikiGraph::open(path: &Path, config: Config)` 시그니처
