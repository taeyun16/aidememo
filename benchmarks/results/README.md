# Benchmark Result Snapshots

Keep only durable, interpretable snapshots in this directory.

- `storage_backend_probe.json` is tracked because every row names its backend
  and the dated storage-backend section in `docs/MEASUREMENTS.md` explains the
  command and execution envelope.
- `performance.json` is generated by `cargo run --release --bin performance`
  but is ignored by default. Its values depend on the default backend, host,
  toolchain, and current commit. Promote a future snapshot only after recording
  that provenance in `docs/MEASUREMENTS.md`.

The unlabelled April 2026 performance snapshot was retired after SQLite became
the default backend; it no longer represented the current product path.
